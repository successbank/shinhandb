🔍 실제 적용된 CSS 스타일 확인 매뉴얼

⚠️ 목적: 현재 http://211.248.112.67:5647/projects 페이지의 썸네일에 실제로 적용된 CSS를 확인하여 크롭 발생 원인을 파악합니다.

📋 확인 방법 (Chrome/Edge 기준)

1단계: 프로젝트 페이지 접속
  • http://211.248.112.67:5647/projects 페이지를 엽니다
  • 썸네일 이미지가 표시된 상태인지 확인합니다
2단계: 개발자 도구 열기
  • F12 키를 누르거나
  • 마우스 우클릭 → "검사" 클릭
3단계: 썸네일 이미지 선택
  • 개발자 도구 왼쪽 상단의 "요소 선택" 아이콘 클릭 (화살표 모양)
  • 또는 Ctrl + Shift + C 단축키 사용
  • 페이지의 썸네일 이미지 위에 마우스를 올려 클릭
4단계: IMG 태그의 CSS 확인
  • 개발자 도구 우측 "Styles" 탭에서 다음 항목들을 확인:

✅ 확인할 CSS 속성들:

  1. object-fit: 현재 값이 무엇인지? (contain / cover / fill 등)
  2. width / height: 설정된 값
  3. max-width / max-height: 설정된 값
  4. display: 현재 값
  5. 어떤 CSS 파일/클래스에서 적용되었는지 (Tailwind? inline style?)
5단계: 부모 DIV 태그의 CSS 확인
  • Elements 탭에서 <img>의 부모 <div> 클릭
  • 다음 항목들을 확인:

✅ 확인할 부모 DIV CSS:

  1. display: flex인지 확인
  2. align-items: center인지 확인
  3. justify-content: center인지 확인
  4. overflow: hidden인지 확인
  5. height: 고정 높이 값 (160px / 256px)
6단계: Computed 탭 확인
  • "Styles" 탭 옆의 "Computed" 탭 클릭
  • 실제로 계산된 최종 CSS 값들을 확인
  • 특히 object-fit 값이 최종적으로 무엇인지 확인

📝 보고할 내용

다음 정보를 확인하여 알려주세요:

  1. IMG 태그의 object-fit 값: ________________
  2. IMG 태그의 width/height 값: ________________
  3. IMG 태그의 max-width/max-height 값: ________________
  4. 부모 DIV의 display 값: ________________
  5. 부모 DIV의 align-items 값: ________________
  6. 어떤 클래스명이 적용되어 있나요? (예: max-w-full, object-contain 등)
💡 스크린샷 캡처 권장:
개발자 도구의 "Styles" 탭과 "Computed" 탭을 각각 스크린샷으로 캡처하면 더욱 정확한 분석이 가능합니다.

🎯 예상되는 시나리오

시나리오 1: object-fit이 contain이 아닌 다른 값
  • 원인: Tailwind 클래스가 올바르게 적용되지 않음
  • 해결: 명시적으로 inline style 또는 !important 추가
시나리오 2: 부모 DIV에 flex 속성이 없음
  • 원인: Tailwind 클래스가 컴파일되지 않음
  • 해결: Next.js 재빌드 필요
시나리오 3: 전역 CSS가 스타일을 덮어씀
  • 원인: globals.css 또는 다른 CSS 파일의 우선순위
  • 해결: CSS 우선순위 조정 또는 !important 사용
