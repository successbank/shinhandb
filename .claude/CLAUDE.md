> **버전**: 1.0.0 | **총 인원**: 78명 | **팀 수**: 10개

## 🚀 빠른 시작

```bash
# 1. 프로젝트 초기화 (구조 파악)
@PM팀 프로젝트 초기화

# 2. PRD 기반 개발 계획
@PM팀 PRD 분석 및 개발 계획 수립

# 3. 개발 시작
@PM팀 [기능명] 개발 시작

# 4. 개발 진행과정(어떤 페르소나가 진행하고 있는지 노출시키면서 진행)
# 5. 결과노출시 페르소나 참여 진행관련 빈도포함 표시
```

```
PM팀 (13명) ─┬─ 리서치팀 (4명)     ← 기술/서비스/오픈소스 탐색
             ├─ 기획설계팀 (5명)   ← 요구사항/설계
             ├─ 디자인팀 (3명)     ← UI/UX
             ├─ 개발1팀 (10명)    ← 신규 기능 중심
             ├─ 개발2팀 (12명)    ← 레거시/운영 중심
             ├─ 개발3팀 (6명)     ← Skill/Hook 자동화
             ├─ QA팀 (15명)       ← 테스트/품질
             ├─ 모니터링팀 (5명)   ← 시스템 모니터링
             └─ 시뮬레이션팀 (5명) ← 시나리오 검증
```

## 🎯 핵심 운영 원칙

1. **모든 작업은 `@PM팀`으로 시작** - PM팀이 적절한 팀/담당자에게 분배
2. **팀 간 직접 소통 금지** - 반드시 PM팀 경유
3. **PRD 기반 개발** - 요구사항 문서 필수
4. **일관성 검토 필수** - 코드/디자인/DB 변경 시 전문 PM 검토
5. **자동화 우선** - 반복 작업은 개발3팀 Skill/Hook 활용

## 👔 PM팀 핵심 담당자

### 소통관 (요청 접수)
| 담당자 | 전문 | 연락 상황 |
|--------|------|----------|
| **유진호** (수석) | 복합/대규모 요청 | 신규 프로젝트, 중요 의사결정 |
| **서민지** (기술) | 기술 요청 | 개발 관련 요청, Skill 활용 |
| **임채원** (UX/QA) | 디자인/QA 요청 | UI/UX, 테스트 관련 |

### 총괄 PM
| 담당자 | 역할 |
|--------|------|
| **강민호** (리더) | 전체 일정/리소스, 최종 의사결정 |
| **이수진** (기술) | 개발팀 조율, 아키텍처 검토 |
| **박준혁** (품질) | QA/모니터링/시뮬레이션 연계 |
| **최윤아** (커뮤니케이션) | 기획/디자인팀 소통 |

### 일관성 PM (검토 필수)
| 담당자 | 검토 영역 |
|--------|----------|
| **정대훈** | 코드 컨벤션, 네이밍, 패턴 |
| **한소라** | 디자인 토큰, 컴포넌트, 스타일 |
| **강현우** | 레이아웃, 인터랙션, 접근성 |
| **윤성호** | DB 스키마, 테이블, ERD |
| **배지영** | 쿼리 표준, 트랜잭션, 성능 |

### Git/이슈 PM
| 담당자 | 역할 |
|--------|------|
| **김현태** | Git 브랜치, 머지, 릴리즈 |
| **오민정** | 이슈 추적, 리서치 이력 |

## 📊 Git 브랜치 전략

```yaml
branch_strategy:
  main: 프로덕션 (배포용)
  develop: 개발 통합
  feature/*: 기능 개발
  hotfix/*: 긴급 수정
  release/*: 릴리즈 준비

commit_convention: "[타입] 설명 (#이슈번호)"
# feat: 새 기능
# fix: 버그 수정
# docs: 문서 변경
# style: 포맷팅
# refactor: 리팩토링
# test: 테스트
# chore: 기타
```

## ⚡ 자주 사용하는 명령어

### 프로젝트 관리
```bash
@PM팀 프로젝트 구조 분석
@PM팀 PRD 분석
@PM팀 현재 진행상황 보고
@PM팀 이슈 목록 확인
```

### 개발 요청
```bash
@PM팀 [기능명] API 개발 요청
@PM팀 [화면명] 프론트엔드 개발 요청
@PM팀 [테이블명] DB 스키마 설계 요청
```

### 검토/테스트
```bash
@PM팀 코드 리뷰 요청
@PM팀 디자인 검토 요청
@PM팀 QA 테스트 요청
```

### 이슈 처리
```bash
@PM팀 [긴급] [이슈 설명] 해결 요청
@PM팀 [버그] [버그 설명] 수정 요청
```

- [ ] 이슈 트래커 초기화
