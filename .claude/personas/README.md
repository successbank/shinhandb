# 페르소나 시스템 사용 가이드

## 📦 설치 방법

### 1. Claude Code에서 프로젝트 초기화
```bash
/init
```

### 2. 페르소나 파일 복사
생성된 `.claude/` 디렉토리에 페르소나 파일들을 복사:

```
.claude/
├── CLAUDE.md           # 메인 설정 파일 내용 복사
└── personas/           # 디렉토리 생성 후 각 팀 파일 복사
    ├── pm팀.md
    ├── 리서치팀.md
    ├── 기획설계팀.md
    ├── 디자인팀.md
    ├── 개발1팀.md
    ├── 개발2팀.md
    ├── 개발3팀.md
    ├── qa.md
    ├── monitoring.md
    └── 시뮬레이션.md
```

### 3. 프로젝트 초기화 실행
```bash
@PM팀 프로젝트 초기화
```

## 🚀 기본 사용법

### 프로젝트 시작
```bash
# 프로젝트 구조 분석
@PM팀 프로젝트 초기화

# PRD 기반 개발 계획
@PM팀 PRD 분석 및 개발 계획 수립

# 개발 시작
@PM팀 [기능명] 개발 시작
```

### 개발 요청
```bash
# API 개발
@PM팀 사용자 인증 API 개발 요청

# 프론트엔드 개발
@PM팀 로그인 화면 프론트엔드 개발 요청

# DB 설계
@PM팀 사용자 테이블 스키마 설계 요청
```

### 검토/테스트
```bash
# 코드 리뷰
@PM팀 코드 리뷰 요청

# QA 테스트
@PM팀 QA 테스트 요청

# 성능 테스트
@PM팀 성능 테스트 요청
```

### 이슈 처리
```bash
# 일반 이슈
@PM팀 [이슈 설명] 해결 요청

# 긴급 이슈
@PM팀 [긴급] [이슈 설명] 해결 요청

# 버그 수정
@PM팀 [버그] [버그 설명] 수정 요청
```

## 👥 팀 호출 방법

### PM팀 경유 (권장)
```bash
@PM팀 [요청사항]
```
→ PM팀이 적절한 팀/담당자에게 자동 분배

### 특정 팀 직접 호출
```bash
@개발1팀 [개발 요청]
@QA팀 [테스트 요청]
@디자인팀 [디자인 요청]
```

### 특정 담당자 호출
```bash
@PM팀.유진호 [소통관 요청]
@개발1팀.김태현 [백엔드 요청]
@QA팀.김정훈 [QA 리드 요청]
```

## 📋 주요 담당자 Quick Reference

| 담당자 | 담당 영역 |
|--------|----------|

| 임채원 | 디자인/QA 요청 |

### 일관성 PM (검토)

|--------|----------|

|--------|-----|------|

## ⚙️ 프로젝트별 커스터마이징

### Git 저장소 설정
CLAUDE.md의 Git 설정 섹션 수정:
```yaml
repository: https://github.com/your-org/your-repo.git
```

### 기술 스택 설정
각 개발팀 파일의 tech_stack 섹션 수정

### 팀 구성 조정
필요에 따라 personas/ 디렉토리의 팀 파일 수정

## 🔧 트러블슈팅

### 페르소나가 응답하지 않는 경우
1. CLAUDE.md 파일이 올바른 위치에 있는지 확인
2. personas/ 디렉토리 구조 확인
3. 파일 인코딩이 UTF-8인지 확인

### 팀 분배가 이상한 경우
```bash
@PM팀 현재 팀 배정 상태 확인
```

### 프로세스 초기화
```bash
@PM팀 프로젝트 초기화
```

## 📌 주의사항

1. **모든 요청은 `@PM팀`으로 시작** - 직접 개발 금지
2. **팀 간 직접 소통 금지** - PM팀 경유 필수
3. **PRD 필수** - 요구사항 문서 없이 개발 시작 금지
4. **일관성 검토 필수** - 코드/디자인/DB 변경 시
