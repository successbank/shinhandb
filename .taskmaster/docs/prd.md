## 1. 개요

### 1.1 문서 목적

본 문서는 신한금융지주 및 신한은행의 광고 자료 통합 관리 플랫폼 개발을 위한 제품 요구사항 명세서입니다. 플랫폼의 목표, 기능 범위, 사용자 역할, 기술 요구사항 등을 정의합니다.

### 1.2 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 신한금융 광고관리 플랫폼 |
| **대상 사용자** | 신한금융지주/신한은행 광고담당자, 임원, 특정인, 클라이언트(광고대행사) |
| **핵심 목표** | 과거 광고 자료(시안, 초안 포함) 검색, 확인, 공유 기능 제공 |
| **주요 특징** | OCR 기반 자동 태그 생성, 회원 유형별 카테고리 세팅, 갤러리/게시물 뷰 |

## 2. 프로젝트 배경 및 목적

### 2.1 배경

신한금융지주와 신한은행은 수년간 다양한 광고 캠페인을 진행해왔으며, 이에 따른 광고 자료(시안, 초안, 최종본)가 대량으로 축적되어 있습니다. 현재 이러한 자료들이 분산 관리되어 필요 시 검색과 활용에 어려움이 있으며, 광고대행사(클라이언트)와의 자료 공유 프로세스도 비효율적인 상황입니다.

### 2.2 목적

1. 광고 자료의 통합 관리 및 체계적 분류
2. OCR 기술을 활용한 자동 태그 생성으로 검색 효율성 극대화
3. 권한 기반 접근 제어로 보안성 확보
4. 클라이언트(광고대행사)와의 효율적인 협업 환경 구축
5. 모든 활동에 대한 감사 로그 관리

### 2.3 기대 효과

- 광고 자료 검색 시간 80% 단축
- 중복 제작 방지로 비용 절감
- 브랜드 일관성 유지 및 품질 관리 강화
- 클라이언트 협업 효율성 향상

## 4. 기능 요구사항

### 4.1 최고관리자 기능

#### 4.1.1 회원 관리

**[FR-001] 회원 생성**
- 신한금융지주/신한은행 회원 생성: 이름, 이메일, 부서, 직급, 소속(지주/은행) 입력
- 클라이언트 생성: 회사명, 담당자명, 이메일, 연락처 입력
- 초기 비밀번호 자동 생성 및 이메일 발송

**[FR-002] 회원 권한 관리**
- 회원별 권한 조회 및 수정
- 계정 활성화/비활성화
- 비밀번호 초기화

#### 4.1.2 카테고리 관리

**[FR-003] 카테고리 설정**
- 1차/2차 카테고리 생성, 수정, 삭제, 순서 변경
- 회원 유형별 기본 카테고리 세팅:
  - **신한금융지주 회원:** CSR, 브랜드, 스포츠, 기타
  - **신한은행 회원:** 브랜드 PR, 상품&서비스, 땡겨요, 기타

#### 4.1.3 모니터링

**[FR-004] 활동 로그 조회**
- 모든 업로드, 수정, 삭제 활동 기록
- 기록 항목: 시간, 계정(사용자 ID), IP 주소, 활동 내용
- 기간별, 사용자별, 활동 유형별 필터링
- 로그 데이터 Excel 내보내기

#### 4.1.4 수정 권한 부여

**[FR-005] 클라이언트 수정 시간 부여**
- 30분 초과된 콘텐츠에 대해 클라이언트에게 수정 시간 부여
- 부여 시간 설정 (30분, 1시간, 2시간, 커스텀)
- 또는 최고관리자가 직접 수정

### 4.2 콘텐츠 관리 기능

#### 4.2.1 업로드

**[FR-006] 파일 업로드**
- 최대 200MB 파일 업로드 지원
- 드래그 앤 드롭 업로드 지원
- 다중 파일 동시 업로드
- 지원 파일 형식: JPG, PNG, GIF, PDF, MP4, MOV, PSD, AI, ZIP
- 업로드 진행률 표시

#### 4.2.2 OCR 기반 태그 자동 생성

**[FR-007] 이미지 문자 인식**
- 이미지 업로드 시 OCR 기능으로 문자 인식
- 인식된 텍스트 기반 자동 태그 생성
- 태그 수동 추가/수정/삭제 기능
- 빈도 높은 태그 자동 추천 (인기 태그)
- 한글, 영문 모두 인식

#### 4.2.3 검색

**[FR-008] 통합 검색**
- 키워드 검색 (제목, 태그, 설명)
- 카테고리 필터
- 날짜 범위 필터
- 파일 유형 필터
- 업로더 필터
- 태그 클릭 검색

### 4.3 보기 형식

**[FR-009] 갤러리 뷰 (기본값)**
- 썸네일 그리드 형식 표시
- 썸네일 하단에 제목 노출
- 그리드 열 수 조정 (2열, 3열, 4열)

**[FR-010] 게시물 뷰**
- 리스트 형식 표시
- 썸네일, 제목, 날짜, 업로더, 태그 표시

### 4.4 공유 기능

**[FR-011] 콘텐츠 공유**
- 목록 상태에서 공유: 선택한 다수 콘텐츠 공유 링크 생성
- 상세페이지 상태에서 공유: 개별 콘텐츠 공유 링크 생성
- 공유 링크 유효기간 설정 (1일, 7일, 30일, 무제한)
- 공유 링크 복사 기능
- 이메일로 공유 링크 발송

### 4.5 마이페이지

**[FR-012] 마이페이지 기능**
- 콘텐츠 보관함 (북마크)
- 개인 메모 작성 (콘텐츠별)
- 내가 업로드한 콘텐츠 목록 (클라이언트)
- 최근 본 콘텐츠
- 계정 정보 수정 (비밀번호 변경)

## 5. 비기능 요구사항

### 5.1 성능

1. 페이지 로딩 시간: 3초 이내
2. 검색 결과 응답 시간: 2초 이내
3. 동시 접속자: 최소 100명 지원
4. 파일 업로드: 200MB 이하 파일 5분 이내 완료

### 5.2 보안

1. HTTPS 통신 필수
2. 비밀번호 암호화 저장 (bcrypt)
3. 세션 타임아웃: 30분 비활동 시 자동 로그아웃
4. 로그인 실패 5회 시 계정 잠금 (30분)
5. IP 기반 접근 로그 기록
6. 금융권 보안 가이드라인 준수

### 5.3 가용성

1. 서비스 가용성: 99.5% 이상
2. 정기 점검: 월 1회, 사전 공지
3. 데이터 백업: 일 1회 자동 백업

### 5.4 호환성

1. 브라우저: Chrome, Safari, Edge, Firefox 최신 2개 버전
2. 반응형 웹: Desktop, Tablet, Mobile 지원
3. 최소 해상도: 1280x720

## 7. 기술 스택 및 아키텍처

### 7.1 기술 스택 (권장)

| 영역 | 기술 | 비고 |
|-----|------|-----|
| Frontend | React.js / Next.js | SSR 지원, SEO 최적화 |
| Backend | Node.js / Express 또는 Python / FastAPI | RESTful API |
| Database | PostgreSQL | 관계형 DB |
| 파일 저장소 | AWS S3 / MinIO | 오브젝트 스토리지 |
| 검색 엔진 | Elasticsearch | 전문 검색, 태그 검색 |
| OCR | Google Cloud Vision / Tesseract | 이미지 문자 인식 |
| 캐시 | Redis | 세션, 캐싱 |
| 인프라 | Docker / Kubernetes | 컨테이너 오케스트레이션 |

### 7.2 시스템 아키텍처

3-tier 아키텍처를 기반으로 프레젠테이션 계층, 비즈니스 로직 계층, 데이터 계층을 분리합니다.

- **프레젠테이션 계층:** React/Next.js 기반 SPA
- **API Gateway:** Nginx / Kong
- **비즈니스 로직:** Node.js/Express 또는 FastAPI 마이크로서비스
- **데이터 계층:** PostgreSQL + Redis + Elasticsearch
- **파일 저장:** 현재 프로젝트 공간

## 8. 데이터베이스 설계

### 8.1 주요 테이블

#### users (사용자)

| 컬럼명 | 타입 | 설명 |
|-------|------|-----|
| id | UUID | PK |
| email | VARCHAR(255) | 이메일 (로그인 ID) |
| password_hash | VARCHAR(255) | 암호화된 비밀번호 |
| name | VARCHAR(100) | 이름 |
| user_type | ENUM | ADMIN, HOLDING, BANK, CLIENT |
| department | VARCHAR(100) | 부서 |
| company_name | VARCHAR(255) | 회사명 (클라이언트용) |
| is_active | BOOLEAN | 활성화 상태 |
| created_at | TIMESTAMP | 생성일시 |

#### contents (콘텐츠)

| 컬럼명 | 타입 | 설명 |
|-------|------|-----|
| id | UUID | PK |
| title | VARCHAR(255) | 제목 |
| description | TEXT | 설명 |
| file_url | VARCHAR(500) | 파일 저장 경로 |
| thumbnail_url | VARCHAR(500) | 썸네일 이미지 경로 |
| file_type | VARCHAR(50) | 파일 유형 |
| file_size | BIGINT | 파일 크기 (bytes) |
| category_id | UUID | FK - categories |
| uploader_id | UUID | FK - users |
| ocr_text | TEXT | OCR 추출 텍스트 |
| editable_until | TIMESTAMP | 수정 가능 시간 |
| created_at | TIMESTAMP | 생성일시 |

#### activity_logs (활동 로그)

| 컬럼명 | 타입 | 설명 |
|-------|------|-----|
| id | UUID | PK |
| user_id | UUID | FK - users |
| action_type | ENUM | UPLOAD, UPDATE, DELETE, LOGIN, LOGOUT |
| target_id | UUID | 대상 콘텐츠 ID |
| ip_address | VARCHAR(45) | 클라이언트 IP |
| details | JSONB | 상세 정보 |
| created_at | TIMESTAMP | 발생일시 |

#### categories (카테고리)

| 컬럼명 | 타입 | 설명 |
|-------|------|-----|

| parent_id | UUID | 상위 카테고리 (NULL이면 1차) |
| target_user_type | ENUM | HOLDING, BANK, ALL |
| sort_order | INTEGER | 정렬 순서 |
| created_at | TIMESTAMP | 생성일시 |

#### tags (태그)

| 컬럼명 | 타입 | 설명 |
|-------|------|-----|
| id | UUID | PK |
| name | VARCHAR(100) | 태그명 |
| usage_count | INTEGER | 사용 횟수 |
| created_at | TIMESTAMP | 생성일시 |

#### content_tags (콘텐츠-태그 연결)

| 컬럼명 | 타입 | 설명 |
|-------|------|-----|
| content_id | UUID | FK - contents |
| tag_id | UUID | FK - tags |

#### bookmarks (북마크/보관함)

| 컬럼명 | 타입 | 설명 |

## 10. 개발 로드맵

### Phase 1: 기반 구축 (4주)

1. 프로젝트 환경 세팅 (개발/스테이징/프로덕션)
2. 데이터베이스 스키마 설계 및 구축
3. 인증/권한 시스템 구현
4. 기본 UI 컴포넌트 개발

### Phase 2: 핵심 기능 (6주)

1. 파일 업로드/다운로드 시스템
2. OCR 연동 및 태그 자동 생성
3. 검색 엔진 연동 (Elasticsearch)
4. 카테고리 관리 기능
5. 갤러리/게시물 뷰 구현

### Phase 3: 부가 기능 (4주)

- 공유 기능 구현
- 마이페이지 기능
- 활동 로그 및 모니터링
- 관리자 대시보드

### Phase 4: 테스트 및 출시 (2주)

- 통합 테스트
- 보안 점검
- 사용자 인수 테스트 (UAT)
- 프로덕션 배포
